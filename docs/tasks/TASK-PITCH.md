# TASK-PITCH · Pitch + Demo 兜底(独立轨)

> 不依赖代码,全程可并行;后期收口。负责人也兼 happy-path QA。

## 独占路径
`docs/pitch/**`(slides 大纲、脚本、roadmap)

## 子任务
- □ **3 分钟脚本**:问题 → 方案 → demo → 市场/商业。开场用一句话:
  "A seller has an idea. Sea Launch AI turns it into a Shopee-ready business case in minutes."
- □ **利润卡话术**:讲清"为什么是 estimated opportunity 不是 guarantee"——
  base 28% / high-return 12% / Committee 给 Watch,这是"可信"的体现。
- □ **Demo 高潮**:吸尘器 → Risk 弹电器/夸大 warning → Committee 给 Watch,把风险变亮点。
- □ **Codex 使用记录**:这是 Codex Hackathon,准备一句"我们怎么用 Codex 把它建出来"。
- □ **slides**:含一页 roadmap,把砍掉的(Lazada / 多地区 / ROI / 实时图)讲成"下一步"。
- □ **录屏兜底**:完整 happy path 录一遍,API 挂了也能放。
- □ **彩排 ≥2 遍**,计时 ≤3 分钟。

## 验收
讲稿定稿;录屏覆盖 Brief→War Room→Board→Studio 全程;计时达标。

## 评委视角收尾句
> Sea Launch AI shows what ecommerce operations look like when AI is not an add-on,
> but the operating team itself.
